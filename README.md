## 🎯 核心特色

### 1. AI友好设计
- **结构化Markdown**: 统一的元数据格式，便于AI解析
- **语义化链接**: 文档间相互引用，构建知识网络
- **标准模板**: 统一的文档模板，保证AI理解一致性

### 2. 完整覆盖开发周期
- ✔️ 项目立项到交付
- ✔️ 需求分析到测试
- ✔️ 代码规范到部署
- ✔️ 团队培养到度量分析

### 3. 实践导向
- 💡 包含大量实施案例
- 📖 提供详细的教程指南
- 🛠️ 实用的工具配置模板
- 🔍 问题排查checklist

## 📖 AI协作最佳实践

### 1. 上下文管理

向AI提供充分的上下文信息：
- 当前工作目录位置
- 相关的技术栈版本
- 项目的架构设计
- 团队编码规范

### 2. Prompt工程

使用结构化的Prompt模板：
- 明确任务目标
- 提供必要约束
- 包含示例代码
- 指定输出格式

详细指南: [Prompt工程原则](04-AI协作指南/04.1-Prompt工程/prompt-principles.md)
