# 新人入职指南

> 欢迎加入我们的团队！本指南帮助你快速融入团队并开始高效工作。

## 入职前准备

### 第一天前一周

**HR会为你准备**:
- [ ] 工作电脑及账号
- [ ] 工牌和门禁权限
- [ ] 办公用品
- [ ] 基本账号开通

**你需要准备**:
- [ ] 了解公司业务和产品
- [ ] 熟悉常用技术栈
- [ ] 准备好相关问题
- [ ] 确认第一天上班时间和地点

### 账号准备清单

| 系统 | 用途 | 申请状态 | 备注 |
|------|------|----------|------|
| 邮箱 | 工作沟通 | ⏳ | 由HR申请 |
| GitLab | 代码管理 | ⏳ | 与邮箱绑定 |
| Jira | 任务管理 | ⏳ | 与邮箱绑定 |
| Confluence | 文档协作 | ⏳ | 与邮箱绑定 |
| VPN | 远程访问 | ⏳ | 需要审批 |
| 云服务 | 开发测试 | ⏳ | AWS/阿里云账号 |

## 第一天安排

### 上午

**9:00-9:30 入职欢迎**
- HR介绍公司文化
- 团队成员介绍
- 办公环境熟悉

**9:30-10:00 设备与环境**
- 领取办公设备
- 账号配置确认
- 开发环境搭建

**10:00-11:00 团队介绍**
- 团队结构介绍
- 项目背景说明
- 伙伴制度介绍

**11:00-12:00 文档阅读**
- [ ] 项目README
- [ ] 技术架构文档
- [ ] 代码规范
- [ ] 团队沟通指南

### 下午

**14:00-15:00 代码仓库**
- 代码仓库访问权限
- 本地环境搭建
- Hello World运行

**15:00-16:00 导师时间**
- 一对一沟通
- 短期目标设定
- 问题解答

**16:00-17:00 第一个任务**
- 领取第一个新手任务
- 熟悉开发流程
- Code Review体验

## 第一周计划

### Day 1-2: 环境搭建

**目标**: 能够本地运行项目并完成基础开发任务

- [ ] 开发环境完全搭建
- [ ] 代码仓库克隆和运行
- [ ] 数据库配置完成
- [ ] 基础配置理解

**测试标准**:
- [ ] 可以本地启动前后端服务
- [ ] 可以访问主要功能页面
- [ ] 可以运行测试用例
- [ ] 可以构建打包

### Day 3-4: 代码熟悉

**目标**: 理解代码结构和主要功能模块

- [ ] 阅读核心模块代码
- [ ] 理解数据流和交互
- [ ] 查看历史问题和解决方案
- [ ] 尝试小Bug修复

**测试标准**:
- [ ] 能画出主要模块关系图
- [ ] 能说明关键业务流程
- [ ] 能定位常见Bug位置
- [ ] 能提交第一个PR

### Day 5: 第一次评审

**目标**: 完成第一个小功能的开发

- [ ] 选择一个Good First Issue
- [ ] 完成开发和测试
- [ ] 提交PR并通过Code Review
- [ ] 功能合并上线

## 常见问题

### 技术问题

**Q: 本地环境搭建遇到困难？**
A:
1. 先查看FAQ文档
2. 询问团队伙伴
3. 联系运维支持
4. 创建Issue跟踪

**Q: 不理解某个业务逻辑？**
A:
1. 先阅读相关文档
2. 查看代码注释
3. 咨询产品经理
4. 请教资深开发

**Q: 代码审查意见很多？**
A:
1. 这是学习的好机会
2. 理解每条意见的原因
3. 不懂就问
4. 总结经验

### 协作问题

**Q: 不知道问谁？**
A: 查看Key Person清单，选择最合适的人

**Q: 提出的想法没有被采纳？**
A:
1. 理解决策原因
2. 持续观察和思考
3. 积累经验后再尝试
4. 关键是用数据说话

**Q: 感觉融入团队困难？**
A:
1. 主动参与团队活动
2. 多向伙伴请教
3. 分享你的经验
4. 保持积极开放

### 个人发展

**Q: 如何规划职业发展？**
A:
1. 前3个月专注技术提升
2. 3-6个月深入业务理解
3. 6个月后考虑专业方向
4. 定期与导师沟通

**Q: 如何平衡工作与生活？**
A:
1. 合理评估任务难度
2. 及时沟通工作进展
3. 遇到困难寻求帮助
4. 保持身心健康
