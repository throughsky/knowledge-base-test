## 1. 数据隐私合规

### 适用法规

| 法规 | 适用范围 | 核心要求 |
|------|----------|----------|
| **GDPR** | 欧盟用户数据 | 数据主体权利、合法处理基础、DPO |
| **CCPA/CPRA** | 加州用户数据 | 知情权、删除权、拒绝销售权 |
| **个人信息保护法** | 中国用户数据 | 告知同意、最小必要、数据本地化 |

### 必须实现的功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **数据查询** | 用户可查询其所有个人数据 | P0 |
| **数据删除** | 用户可请求删除其数据 | P0 |
| **数据导出** | 用户可导出其数据(可移植性) | P1 |
| **同意管理** | 记录用户同意的范围和时间 | P0 |
| **数据访问日志** | 记录谁访问了用户数据 | P0 |

### 数据保留策略

```yaml
data_retention:
  user_account:
    active: "永久（直到用户删除）"
    deleted: "30天后彻底删除"
  transaction_records:
    retention: "7年（财务合规）"
  access_logs:
    retention: "1年"
  marketing_data:
    retention: "2年（无活动后）"
```

## 2. 行业特定合规

### 金融行业

| 标准 | 要求 |
|------|------|
| **PCI-DSS** | 支付卡数据安全 |
| **SOX** | 财务报告内控 |
| **反洗钱(AML)** | 可疑交易监控 |

### 医疗健康

| 标准 | 要求 |
|------|------|
| **HIPAA** | 健康信息保护(美国) |
| **医疗器械法规** | 软件医疗器械(SaMD) |

## 3. 技术合规检查清单

### 数据处理
- [ ] 个人数据收集有明确法律基础
- [ ] 用户已知情并同意数据处理
- [ ] 数据处理遵循最小必要原则
- [ ] 敏感数据有额外保护措施

### 数据主体权利
- [ ] 实现数据查询接口
- [ ] 实现数据删除接口
- [ ] 实现数据导出接口
- [ ] 同意撤回机制可用

### 数据安全
- [ ] 传输加密(TLS)
- [ ] 存储加密
- [ ] 访问控制
- [ ] 审计日志

### 跨境传输
- [ ] 识别数据跨境场景
- [ ] 有合法传输机制(SCCs/BCRs)
- [ ] 数据本地化要求已满足
