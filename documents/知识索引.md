### 3. TDD 测试驱动开发
**完整的 TDD 实践指南**

| 文档 | 内容概要 | 学习目标 |
|------|----------|----------|
| [TDD 实践指南](./03-TDD测试驱动开发/TDD实践指南.md) | TDD 核心理念、工作流程、实例演示 | 掌握测试驱动开发方法 |

**核心内容**:
- TDD 三定律
- 红灯-绿灯-重构循环
- 测试金字塔 (单元/集成/E2E)
- AAA 测试模式 (Arrange-Act-Assert)
- Mock 和 Stub 使用
- 实际项目示例

**关键实践**:
```typescript
// 1. 先写失败的测试
it('should calculate total price', () => {
  const cart = new ShoppingCart();
  cart.addItem({ price: 10, quantity: 2 });
  expect(cart.totalPrice).toBe(20);
});

// 2. 编写最小实现代码
class ShoppingCart {
  constructor(public totalPrice = 0) {}
  addItem(item: any) {
    this.totalPrice += item.price * item.quantity;
  }
}

// 3. 重构优化
```

### 4. 架构设计规范
**架构决策和技术选型指南**

| 文档 | 内容概要 | 适用角色 |
|------|----------|----------|
| [架构设计原则](./04-架构设计规范/架构设计原则.md) | SOLID 原则、设计模式、微服务架构 | 架构师、高级开发 |

**主要内容**:
- **SOLID 原则**: 详细示例和实践指导
  - SRP: 单一职责原则
  - OCP: 开闭原则
  - LSP: 里氏替换原则
  - ISP: 接口隔离原则
  - DIP: 依赖倒置原则
- **模块设计**: 高内聚、低耦合的实现
- **架构模式**: MVC、六边形、CQRS、事件溯源
- **微服务设计**: 服务拆分、通信方式、数据一致性
- **性能优化**: 缓存策略、数据库优化

**架构决策记录 (ADR)**:
```markdown
# ADR-001: 使用微服务架构

## 背景
单体应用规模扩大，团队增长...

## 决策
拆分为微服务：用户、订单、商品服务

## 后果
积极: 独立部署、技术异构
消极: 复杂性增加
```

### 5. 技术栈规范
**核心技术选型和工具使用**

| 文档 | 内容概要 | 技术维度 |
|------|----------|----------|
| [技术栈规范](./05-技术栈规范/技术栈规范.md) | 前端/后端/数据库/工具链 | 全栈技术选型 |

**前端技术栈**:
- **框架**: React 18+ / Next.js 14
- **语言**: TypeScript 5.3+
- **UI**: Ant Design 5.x / Tailwind CSS
- **构建**: Vite 5.x
- **测试**: Jest + React Testing Library

**后端技术栈**:
- **Node.js**: NestJS 10+ (企业级框架)
- **Python**: FastAPI (高性能)
- **数据库**: PostgreSQL 15+ (首选)
- **缓存**: Redis 7+
- **消息队列**: RabbitMQ / Kafka

**基础设施**:
- **容器化**: Docker
- **CI/CD**: GitHub Actions / GitLab CI
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack

**决策矩阵**:
| 技术 | 适用场景 | 优势 |
|------|----------|------|
| Next.js | SSR/全栈应用 | SEO友好、性能好 |
| NestJS | 企业级API | TypeScript、IOC容器 |
| PostgreSQL | 关系型数据 | 功能丰富、稳定 |
| Redis | 缓存/队列 | 高性能、多功能 |

## 🎯 使用指南

### 新功能开发流程

```

    ↓
步骤2: 编写SDD规范文档
    ↓

    ↓
步骤7: 合并部署
```
