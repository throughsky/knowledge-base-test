## 核心理念

本精简版聚焦于 **Speckit SDD 规范驱动** 和 **TDD 测试驱动** 两个核心实践，帮助团队建立高质量的代码交付标准。

### 两大核心实践

1. **SDD 规范驱动开发 (Specification-Driven Development)**
   - 先写设计规范，再写实现代码
   - 规范即文档，代码即实现
   - AI 可以直接理解规范并生成代码

2. **TDD 测试驱动开发 (Test-Driven Development)**
   - 先写测试，再写实现
   - 红灯-绿灯-重构循环
   - 保证代码质量和可维护性

## 快速开始

### 新功能开发流程

```
1. 需求分析 → 创建用户故事
2. 设计规范 → 编写 SDD 文档
3. 测试先行 → 编写测试用例
4. AI 生成 → AI 根据规范生成代码
5. 验证实现 → 测试通过 + Code Review
6. 重构优化 → 持续改进代码质量
```

### 文档使用指南

1. **从章程开始**: 了解团队开发原则
2. **学习 SDD**: 掌握规范编写方法
3. **实践 TDD**: 建立测试驱动思维
4. **参考架构**: 遵循架构设计规范
5. **使用技术栈**: 选择合适的技术方案
