## 联系规则

### 联系时间

| 时间 | 联系渠道 | 响应时间 | 备注 |
|------|----------|----------|------|
| 工作日 9:00-18:00 | 任意 | 2小时内 | 优先工作群 |
| 工作日 18:00-22:00 | 电话/紧急 | 30分钟内 | 紧急情况 |
| 周末/节假日 | 紧急热线 | 1小时内 | 仅限P0/P1故障 |
| 深夜 | 仅限生产故障 | 立即 | |

### 联系优先级

**问题等级**:
- **P0 (紧急)**: 生产系统完全不可用
  - 联系: 电话 + 工作群@all
  - 通知: 项目发起人、技术负责人、运维负责人

- **P1 (高)**: 核心功能不可用
  - 联系: 电话 + 工作群@相关负责人
  - 通知: 技术负责人、产品经理

- **P2 (中)**: 非核心功能问题
  - 联系: 工作群@相关负责人
  - 通知: 相关开发人员

- **P3 (低)**: 一般性问题、咨询
  - 联系: 工作群/邮件
  - 通知: 直属负责人

### 非工作时间

**紧急联系**:
```
1. 查看服务值班表，联系值班人员
2. 如无响应，联系团队负责人
3. 仍无响应，联系项目经理
4. 重大事件联系项目发起人
```

**非紧急情况**:
- 发送邮件，工作日处理
- 在工作群留言，次日回复
- 记录在问题跟踪系统中

## 联系规范

### 邮件规范

**主题格式**:
```
[项目名][类型][主题]
示例: [用户中心][Bug]登录接口返回500错误

类型可以是: [需求][Bug][优化][讨论][通知]
```

**收件人**:
- To: 需要采取行动的直接责任人
- Cc: 需要知晓但不需行动的间接人员
- Bcc: 保密抄送

### 会议邀请

**必需包含**:
- 会议主题
- 会议议程
- 预期产出
- 会前准备材料

**提前时间**:
- 内部会议: 提前24小时
- 跨团队会议: 提前48小时
- 客户会议: 提前1周

### 电话沟通

**适用场景**:
- 紧急情况
- 需要快速决策
- 复杂问题解释
- 建立人际关系

**注意**:
- 工作时间外慎用
- 提前确认对方是否方便
- 重要内容邮件确认
